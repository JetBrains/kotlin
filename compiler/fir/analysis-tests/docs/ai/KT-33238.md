# KT-33238

## Summary
False positive "USELESS_CAST" in delegate init block with internal type value cast to public type (reported for K1 new inference).

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt33238.kt`

## Observations
The test uses the simplified reproducer from the ticket comments: a delegate property where a `private val` of an `internal` type is cast to a `public` type inside the delegate initializer lambda. In K1's new type inference, this cast was incorrectly flagged as `USELESS_CAST` (false positive), because the cast is necessary for cross-module visibility — without it, the type parameter `T` would be inferred as `Internal`, which is not accessible from other modules.

In K2/FIR, the test compiles without any `USELESS_CAST` diagnostic. The compiler correctly handles the delegate init block with an internal type cast to a public type. All test variants passed (PhasedJvmDiagnosticLightTreeTestGenerated, PhasedJvmDiagnosticPsiTestGenerated, FirLightTreeDiagnosticsWithLatestLanguageVersionTestGenerated, and LL tests). The bug described in the ticket is not present in the K2/FIR compiler.
