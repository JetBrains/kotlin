# KT-28697

## Summary
Report warning when the class property is shadowed by a local variable.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt28697.kt`

## Observations
The issue requests a warning/diagnostic when a local variable inside a member function shadows a
class-level property with the same name. Currently, the Kotlin compiler does not emit any warning
in this case.

The test was created with `RUN_PIPELINE_TILL: BACKEND` (code compiles without errors). The test
confirms that no `NAME_SHADOWING` or similar diagnostic is produced for the `val name = "david"`
declaration that shadows `Foobar.name`. This matches the reported behavior — the feature is not
yet implemented.

The first run with `--update-test-data` did not auto-write the `GENERATED_FIR_TAGS` footer, so it
was added manually based on the expected content observed in the test XML report. The second run
(verification) passed successfully with `BUILD SUCCESSFUL`.
