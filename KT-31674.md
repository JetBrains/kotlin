# KT-31674

## Summary
Wrong upper bound for star projection with recursive upper bound: `fun f(x: B<*>) = x.out.out` failed to compile when `B<T : A<T>>`.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt31674.kt`

## Observations
The test uses `RUN_PIPELINE_TILL: BACKEND` with no diagnostic markers, meaning the code compiles
without errors in FIR (K2). The ticket described that `fun f(x: B<*>) = x.out.out` failed in
both old and new (at the time) type inference:

- Old inference: "type mismatch: inferred type is A<*> but A<out A<*>> was expected"
- New inference: "unresolved reference: out"

In the current FIR frontend, the code compiles cleanly — the star projection upper bound
constraint `T : A<T>` is correctly propagated so that `x.out` has type `A<*>` and `x.out.out`
resolves to `Any?` without errors. The bug is no longer reproducible in FIR/K2.

All test variants passed (PhasedJvmDiagnosticLightTreeTestGenerated, PhasedJvmDiagnosticPsiTestGenerated,
and all LL* tests).
