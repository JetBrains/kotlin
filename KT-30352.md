# KT-30352

## Summary
Lambda type inference should not be necessary when only a single overload accepts a lambda

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt30352.kt`

## Observations
The simplified reproducer from the ticket comments:
```kotlin
class A<T>
class B {
    fun <T> foo(a: A<T>, value: T) {}
    fun <T> foo(a: A<T>, value: (A<T>) -> T) {}
}
fun usage(b: B) {
    b.foo(A(), 10)
    b.foo(A<Long>()) { 10L }
}
```

In K1 (1.3.21, 1.7.10), calling `b.foo(A<Long>()) { 10L }` would fail with:
> Cannot choose among the following candidates without completing type inference

In K2/FIR, the test compiles with `RUN_PIPELINE_TILL: BACKEND` and no error diagnostics —
all 7 test variants (LightTree, PSI, LL API, etc.) pass cleanly. The K2 compiler correctly
resolves the lambda overload without requiring explicit type inference. The issue is not
reproduced in K2.
