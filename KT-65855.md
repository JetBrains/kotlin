# KT-65855

## Summary
Modality of intersection override of interop properties: K2 visibility inference sets UNKNOWN modality
for intersection override of two FINAL ObjC protocol properties instead of FINAL, breaking klib serialization.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt65855.kt`

## Observations
This is a Kotlin/Native-specific issue that involves ObjC CInterop. The bug occurs when:
1. ObjC protocol properties are marked as FINAL by the CInterop layer (not possible in pure Kotlin interfaces)
2. A class inherits from two such protocols with the same FINAL property
3. K2's new visibility inference (KT-63741) sets UNKNOWN modality for the intersection override

The test in `compiler/fir/analysis-tests/testData/resolve/problems/kt65855.kt` simulates the general
pattern with regular Kotlin interfaces (where properties are ABSTRACT by default), not FINAL ObjC
properties. In this simulated scenario, the test compiles cleanly with no UNKNOWN modality issues.

The actual bug requires K/N CInterop with ObjC platform libraries (UIKit's `UIDynamicItemProtocol.bounds`
and `UICoordinateSpaceProtocol.bounds`) to reproduce. The original reproducer is the K/N test `kt40426.kt`
which requires running on an iOS simulator target (`ios_simulator_arm64`) — not testable in this
FIR analysis test infrastructure.

The FIR analysis test passes cleanly, showing that standard Kotlin intersection overrides (of abstract
properties) are handled correctly. The K/N-specific UNKNOWN modality issue for FINAL ObjC properties
cannot be reproduced in pure FIR analysis tests.
