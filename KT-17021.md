# KT-17021

## Summary
`IMPLICIT_CAST_TO_ANY` isn't reported for lambda result when an if-expression with Int/Unit branches is the last expression of a lambda.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt17021.kt`

## Observations
The test reproduces the issue. In K2/FIR, `IMPLICIT_CAST_TO_ANY` is not reported for either case:

- **`val f`** (lambda result): The `if` expression with `Int` / `Unit` branches is the last expression of the lambda, so the lambda has type `() -> Any`. No `IMPLICIT_CAST_TO_ANY` warning is emitted — this is the original bug.

- **`val g`** (explicit variable): In K1, `IMPLICIT_CAST_TO_ANY` was correctly reported for `val x = if (someCondition()) { 42 } else { }`. In K2/FIR, however, this warning is also absent.

The test passes cleanly without any diagnostic markers, matching the current (broken) compiler output where `IMPLICIT_CAST_TO_ANY` is never emitted. The first run with `--update-test-data` generated `GENERATED_FIR_TAGS` and both runs completed successfully (BUILD SUCCESSFUL).

The bug is confirmed present in K2/FIR: the inconsistency from K1 (missing warning for lambda result) persists, and additionally the `IMPLICIT_CAST_TO_ANY` diagnostic is no longer emitted even for the explicit-variable case.
