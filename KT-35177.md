# KT-35177

## Summary
`WHEN_ENUM_CAN_BE_NULL_IN_JAVA` compiler warning incorrectly fires for `when` expression whose subject is `Optional.ofNullable(...).orElse(...)`, even though `orElse` guarantees a non-null result.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt35177.kt`

## Observations
The test compiles cleanly under K2/FIR with `RUN_PIPELINE_TILL: BACKEND` and no diagnostic markers — confirming that the K2 compiler does **not** emit `WHEN_ENUM_CAN_BE_NULL_IN_JAVA` for a `when` whose subject is `Optional.ofNullable(color).orElse(Color.BLUE)`. The bug described in the ticket (a spurious null-safety warning for an exhaustive `when` over a result that cannot be null) is not present in the FIR frontend. The issue was a false positive in the K1 (legacy) frontend only; it is already resolved in K2.
