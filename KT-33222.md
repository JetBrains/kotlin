# KT-33222

## Summary
Editing hangs while entering arguments to highly overloaded Java method (assertThat from assertj with ~87 overloads inside a `with` block).

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt33222.kt`

## Observations
The ticket describes an IDE performance problem in K1/old-inference where entering argument list to a highly
overloaded Java method (assertThat from assertj, ~87 overloads) inside a `with` block caused the IDE to hang.

A FIR (K2) analysis test was created simulating the scenario with 26 overloads of `assertThat` called inside
a `with(Assert)` block, including calls with different argument types (Int, String, Double, Boolean, IntArray,
List<String>).

During test development:
1. Without `WITH_STDLIB`, the `with` call produced `CANNOT_INFER_PARAMETER_TYPE` because `listOf` was unresolved,
   preventing `with`'s return type parameter `R` from being inferred.
2. After adding `// WITH_STDLIB`, the code compiled cleanly to BACKEND with no errors.

The FIR (K2) compiler handles this case correctly without errors — it resolves the overloaded calls and infers
types properly. The performance problem described in the ticket appears to be specific to the K1 frontend/IDE
analysis and is not observable as a correctness issue in FIR.
