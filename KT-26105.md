# KT-26105

## Summary
"Name shadowed" warning doesn't work for nested lambdas — only the immediately enclosing scope is checked.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt26105.kt`

## Observations
The test contains two cases:
1. `f()` — a direct lambda parameter `name` that shadows the function parameter `name` (K1 DOES warn here)
2. `g()` — a nested lambda parameter `name` that shadows the function parameter `name` through two levels of lambdas (K1 does NOT warn here — the bug)

When running with `--update-test-data`, the test framework removed the `<!NAME_SHADOWING!>name<!>` diagnostic marker from `f()`, indicating that K2/FIR does not produce a `NAME_SHADOWING` warning for either case. The test passes cleanly with no diagnostic markers, confirming that neither the direct shadow (case 1) nor the nested shadow (case 2) produces a warning in FIR.

The bug is reproduced in K2/FIR in an even stronger form: while K1 at least warned for the direct shadow in `f()`, K2/FIR produces no name shadowing warning at all for lambda parameters in either case.
