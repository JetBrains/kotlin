# KT-27831

## Summary
False positive USELESS_CAST diagnostic when casting a raw Java generic type to a non-raw Kotlin parameterized type.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt27831.kt`

## Observations
The FIR compiler reports `USELESS_CAST` on `it as Inlay<*>?` when `it` has type `Inlay<(raw) Any!>!`
(obtained from iterating over a `List<Inlay>` returned by a Java method using a raw type). The cast
is not useless — it changes the type from raw to a parameterized Kotlin type — but the compiler
incorrectly treats it as redundant.

The `<!USELESS_CAST!>` diagnostic marker was confirmed by the test framework on the first run
with `--update-test-data`, demonstrating the bug is still present in the FIR frontend.
