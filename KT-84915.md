# KT-84915

## Summary
Internal compiler error "Expected expression 'FirAnonymousFunctionExpressionImpl' to be resolved" when using a lambda in the RHS of an equality operator inside a PCLA (Polymorphic Callable Lambda Analysis) builder lambda.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt84915.kt`

## Observations
The test consistently fails with an internal compiler exception in both `--update-test-data` and clean runs.

The exception thrown is:
```
org.jetbrains.kotlin.util.FileAnalysisException: While analysing /kt84915.kt:16:9:
  org.jetbrains.kotlin.utils.exceptions.KotlinIllegalArgumentExceptionWithAttachments:
    Expected expression 'FirAnonymousFunctionExpressionImpl' to be resolved
```

Stack trace shows the crash occurs in:
- `FirDataFlowAnalyzer.exitEqualityOperatorCall` — when the DFA tries to get the resolved type of the anonymous function expression after `t == {}` is processed
- `FirExpressionsResolveTransformer.transformEqualityOperatorCall` — the equality operator call transformation triggers the DFA, but the lambda `{}` hasn't been resolved yet at that point

The bug is reproduced by the code in `buildBox { t == {} }` where `t: T` (type variable bounded by `Any`) is compared with an anonymous function expression `{}`. The lambda ends up unresolved when the data flow analyzer processes the equality call, causing the assertion to fail.

This is a PCLA (Polymorphic Callable Lambda Analysis) scenario — the lambda `{}` used as the RHS of `==` inside a builder lambda triggers a state where `FirAnonymousFunctionExpressionImpl` is not yet resolved when `exitEqualityOperatorCall` is called.
