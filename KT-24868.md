# KT-24868

## Summary
Spurious "Restricted suspending functions can only invoke member or extension suspending functions on their restricted coroutine scope" when calling a `suspend` extension function literal using the `function(receiver, arg0, arg1)` form inside a `@RestrictsSuspension` block.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt24868.kt`

## Observations
The test reproduces the scenario from the ticket: calling `Foo.f(this, 1, 10)` inside a `sequence` block (which uses `@RestrictsSuspension` via `SequenceScope`), where `f` is a `suspend SequenceScope<Int>.(Int, Int) -> Unit` property accessed via an object path.

In K2/FIR, the code compiles without any diagnostics — both the `Foo.f(this, 1, 10)` call (the previously-erroring form) and the `this.f(1, 10)` form (the workaround) compile cleanly. The spurious restricted suspension error described in the ticket is no longer produced by the compiler, indicating the bug has been fixed in K2.
