# KT-11397

## Summary
Improve diagnostic message for INACCESSIBLE_TYPE — the K1 error showed a confusing self-referential message; K2 does not have this diagnostic at all.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt11397.kt`

## Observations
The `INACCESSIBLE_TYPE` diagnostic no longer exists in K2/FIR (confirmed by the comment "No such diagnostic in K2" on the ticket).

Instead, K2 emits clearer, separate diagnostics for related scenarios:
- `EXPOSED_FUNCTION_RETURN_TYPE` — when a public function exposes a protected return type
- `INFERRED_INVISIBLE_RETURN_TYPE_WARNING` — when a variable's inferred type is invisible
- `INVISIBLE_REFERENCE` — when a member of an invisible type is called

The test passes successfully with `// RUN_PIPELINE_TILL: FRONTEND` (code has errors) and the diagnostic markers match K2's actual output. The original confusing INACCESSIBLE_TYPE message from K1 is gone; the issue is effectively resolved in K2 by replacing it with more targeted diagnostics.
