# KT-10741

## Summary
Intersection types in type-inference: support for intersection types in the Kotlin type-inference algorithm.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt10741.kt`

## Observations
The test exercises two intersection type inference scenarios using smart casts:

1. `process<T : A>(t: T): T` called with `x` of smart-cast type `A & B` — the result `result.bar()` compiles successfully, meaning T is correctly inferred as the intersection type `A & B`.
2. `identity<T>(x: T): T` called with `x` of smart-cast type `A & B` — both `result.foo()` and `result.bar()` compile successfully.

The test compiles cleanly to `BACKEND` with no errors or diagnostics. The GENERATED_FIR_TAGS include `intersectionType` and `smartcast`, confirming the FIR compiler correctly handles intersection types in type inference. The behavior described in the ticket (support for intersection types in type-inference) is working correctly in K2/FIR.
