# KT-44008

## Summary
Improve diagnostic messages with `TypeVariable(T)` in new inference — error messages should use `???` instead of internal type variable names like `TypeVariable(R)`.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt44008.kt`

## Observations
The test uses the reproducer from the ticket:
```kotlin
fun f(s: List<String>) {
    s.flatMap { it }
}
```

The original ticket reported that the new inference emitted:
```
error: type mismatch: inferred type is String but Iterable<TypeVariable(R)> was expected
```

The current compiler (K2/FIR) emits instead:
```
/kt44008.kt:(210,217): error: Cannot infer type for type parameter 'R'. Specify it explicitly.
/kt44008.kt:(220,222): error: Return type mismatch: expected 'Iterable<uninferred R (of fun <T, R> Iterable<T>.flatMap)>', actual 'String'.
```

The specific string `TypeVariable(R)` no longer appears in the error message. The message now reads `uninferred R (of fun <T, R> Iterable<T>.flatMap)` which is more informative (it identifies which function's type parameter `R` refers to), though it still does not use the `???` notation requested in the ticket. The test passes cleanly asserting `CANNOT_INFER_PARAMETER_TYPE` on `flatMap` and `RETURN_TYPE_MISMATCH` on `it`.
