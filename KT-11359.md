# KT-11359

## Summary
Overloaded method resolution with generics is incorrect — Kotlin incorrectly reported ambiguity when calling an overloaded method where one overload takes `T` and another takes `A<T>`.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt11359.kt`

## Observations
The test code:
```kotlin
class A<T> {
    fun bar(t: T) {}
    fun bar(a: A<T>) {}
}

fun test(a: A<Any>, b: A<Any>) {
    a.bar(b)
}
```

The test ran with `// RUN_PIPELINE_TILL: BACKEND` and passed cleanly without any `OVERLOAD_RESOLUTION_AMBIGUITY` diagnostic. The FIR compiler (K2) correctly resolves `a.bar(b)` to `bar(a: A<T>)` since `A<Any>` is more specific than `Any`. The bug described in the ticket (ambiguity reported by K1) is no longer present in FIR.
