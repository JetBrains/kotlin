# KT-49035

## Summary
`@kotlin.internal.Exact` type inference error message has reversed expected and actual types: it says "inferred type is Any but String was expected" instead of "inferred type is String but Any was expected".

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt49035.kt`

## Observations
The reproducer code calls `foo<Any>("")` where `foo` has a parameter annotated with `@kotlin.internal.Exact`. The compiler produces an `ARGUMENT_TYPE_MISMATCH` diagnostic on the string literal `""`.

The original issue (K1) reported `TYPE_MISMATCH` with a confusingly reversed message: "inferred type is Any but String was expected" — when in reality, `String` is the inferred/actual type and `Any` is the expected (exact) type.

In K2/FIR, the diagnostic is renamed to `ARGUMENT_TYPE_MISMATCH` and it fires on the argument `""`. The test also reports `ERROR_SUPPRESSION` on `"INVISIBLE_REFERENCE"` in the suppress annotation. The bug scenario is present: the compiler reports an error for this code. The test passes by asserting the current (K2) diagnostic behavior.
