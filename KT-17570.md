# KT-17570

## Summary
Avoid creating FlexibleType instances if one of the bounds is error type (unexpected USELESS_ELVIS on `J.getAny() ?: J()` when T cannot be inferred)

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt17570.kt`

## Observations
The test reproduces the scenario from the ticket: a Java generic method `J.getAny()` with unresolved type parameter `T` is used in an elvis expression `J.getAny() ?: J()`. In K1, this caused an unexpected `USELESS_ELVIS` diagnostic because `KotlinTypeFactory.flexibleType` created a flexible type from two error types (non-null and nullable "uninferred parameter T"), and `FlexibleType.isError` delegated to the lower bound, returning `true`.

In K2/FIR, the test compiles successfully to `BACKEND` with no diagnostics. The `USELESS_ELVIS` warning is NOT produced, meaning K2/FIR correctly handles flexible types with uninferred type parameters and does not exhibit the erroneous behavior described in the ticket.

The test passed cleanly on the second run after manually adding the `GENERATED_FIR_TAGS` comment (the first run with `--update-test-data` had a race condition between FirLightTree and PhasedJvmDiagnostic test runners updating the file concurrently).
