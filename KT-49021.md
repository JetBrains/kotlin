# KT-49021

## Summary
Investigate rules of computations under `NewResolvedCallImpl.getExpectedTypeForSamConvertedArgument`, specifically when it could return `null` during fun interface (SAM) conversion.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt49021.kt`

## Observations
The test exercises SAM conversion for fun interfaces in two scenarios:
1. A direct lambda passed where a fun interface is expected
2. A Kotlin function type variable (`() -> Int`) passed where a fun interface is expected (SAM conversion from function type)

The ticket is an investigation task about K1's `NewResolvedCallImpl.getExpectedTypeForSamConvertedArgument` returning `null`, with the description noting that reverting the commit "Psi2ir: fix function type check for fun interface conversion" would reproduce the problem. The FIR/K2 test passes cleanly (no errors), confirming that both SAM conversion scenarios are handled correctly in K2. The issue is present in K1 (the test documents the relevant behavior and confirms K2 resolves both patterns correctly).
