# KT-14192

## Summary
No warning on platform class usage in static method call (`Integer.parseInt("42")`) and class reference (`Integer::class`).

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt14192.kt`

## Observations
The test confirms the bug is still present in FIR (K2 frontend):

1. `val x: Integer? = null` — correctly produces `PLATFORM_CLASS_MAPPED_TO_KOTLIN` warning
2. `Integer.parseInt("42")` — no `PLATFORM_CLASS_MAPPED_TO_KOTLIN` warning emitted (bug)
3. `val y = Integer::class` — no `PLATFORM_CLASS_MAPPED_TO_KOTLIN` warning emitted (bug)

The diagnostic marker `<!PLATFORM_CLASS_MAPPED_TO_KOTLIN!>Integer<!>?` is only present for the type annotation usage (case 1). No markers appear for the static method call (case 2) and class reference (case 3), confirming the FIR checker does not inspect platform class usages in those positions.

The test passes with the current (broken) behavior — the diagnostic markers reflect the absence of warnings where they should appear.
