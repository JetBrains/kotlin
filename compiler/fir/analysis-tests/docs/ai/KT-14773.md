# KT-14773

## Summary
USELESS_ELVIS warning is not reported in presence of a "type mismatch" error

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt14773.kt`

## Observations
The issue reported that when `it.first to it.second ?: "foo"` causes a type mismatch (because `?:` applies
to the whole `Pair` expression, not just `it.second`), the `USELESS_ELVIS` warning was suppressed and only
the type error was shown in K1.

In K2/FIR, both diagnostics are correctly reported simultaneously:
- `RETURN_TYPE_MISMATCH` on the entire lambda body expression
- `USELESS_ELVIS` on the `?: "foo"` part (since `it.first to it.second` produces a non-nullable `Pair`)

This means the issue is **fixed** in the K2 compiler — it now shows both the type error and the useless
elvis warning, providing the programmer with the helpful hint about incorrect operator precedence.
