# KT-34004

## Summary
False-positive USELESS_CAST warning emitted for `a as B` in a local property initializer when `a` is already smart-cast to `B`, even though the cast actually changes the inferred type of the local variable.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt34004.kt`

## Observations
K2/FIR still produces the `USELESS_CAST` diagnostic for `a as B` in:
```kotlin
interface A
class B : A
fun test(a: A) {
    if (a !is B) return
    var b = a as B  // USELESS_CAST reported here
}
```
The cast is considered useless because `a` is smart-cast to `B` at that point. However, the cast is NOT useless: without it, `var b = a` would infer `b: A` (the declared type of `a`), while `var b = a as B` infers `b: B`. The test passes with `<!USELESS_CAST!>as B<!>` confirming the false-positive is present in K2/FIR.
