# KT-18340

## Summary
False negative IMPLICIT_CAST_TO_ANY with elvis operator: `(null as String?) ?: 1` does not produce an `IMPLICIT_CAST_TO_ANY` warning.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt18340.kt`

## Observations
The test contains both cases from the ticket:
- `val g = if (true) 1 else 2.0` — in K1 (before 1.7.10) this produced `IMPLICIT_CAST_TO_ANY`; in K2/FIR no warning is generated
- `val h = (null as String?) ?: 1` — the bug case; no `IMPLICIT_CAST_TO_ANY` warning is generated

The test passes without any inline diagnostic markers, confirming that in the current FIR/K2 compiler neither case produces an `IMPLICIT_CAST_TO_ANY` warning. The `?:` (elvis) case still has no warning when it arguably should warn that both `String` and `Int` branches are implicitly cast to `Any`. The ticket remains "Open" with the desired behavior (warning on the elvis case) not yet implemented.
