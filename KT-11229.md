# KT-11229

## Summary
Provide warning 'implicitly cast to Any?' for Elvis branches (asymmetry with `if` expression in K1).

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt11229.kt`

## Observations
The ticket reported that in K1, `if (x != null) x else y` produced an `IMPLICIT_CAST_TO_ANY` warning
when `x: A?` and `y: B?`, but the equivalent Elvis expression `x ?: y` did not.

In K2/FIR, there is no `IMPLICIT_CAST_TO_ANY` warning at all — neither the `if` expression nor the
Elvis expression produce the warning. The test compiles cleanly with `RUN_PIPELINE_TILL: BACKEND` and
no diagnostic markers, confirming the asymmetry is gone. The issue is marked "Obsolete" in YouTrack
with the comment "There is no such warning in K2 at all."

The bug (asymmetry between `if` and Elvis) is no longer present in K2, so `repro: false`.
