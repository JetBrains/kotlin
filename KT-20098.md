# KT-20098

## Summary
AE in Kotlin script: "Recursion detected on input" after referring to itself when declaring Pair

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt20098.kt`

## Observations
The ticket reports that `val (a, b) = a` in a Kotlin script causes an `AssertionError: Recursion detected on input: a` in K1 (the legacy frontend). The error occurred because the destructuring declaration's RHS referred to the variable being defined (`a`), causing infinite recursion in K1's lazy resolution.

In K2/FIR, the same code does not crash. Instead, the compiler reports proper diagnostics:
- `SYNTAX` error on `(a, b)` — because top-level destructuring declarations are not allowed in regular `.kt` files (only in scripts and local scopes)
- `UNRESOLVED_REFERENCE` on `a` — because `a` is used before it is defined

The test passes cleanly with these expected diagnostics. The `AssertionError` / "Recursion detected" bug present in K1 is not reproduced in K2/FIR: FIR handles the self-referential destructuring gracefully without crashing.
