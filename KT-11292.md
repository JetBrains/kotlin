# KT-11292

## Summary
Call resolver exceptions — the compiler crashes with internal errors/stack overflows instead of producing proper diagnostics for invalid code patterns involving cyclic delegation.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt11292.kt`

## Observations
The test reproduces a sub-case of KT-11292 (specifically related to KT-30062 and KT-11493): a class delegating to itself via `class B : B by foo {}` causes a `java.lang.StackOverflowError` in the FIR compiler.

The infinite recursion occurs in the FIR scope resolution machinery:
- `FirDelegatedMemberScope.buildScope()` calls `ScopeUtils.scope()`
- Which tries to get the scope of `B`'s supertype (which is `B` itself)
- Which calls `FirDelegatedMemberScope.getCallableNames()`
- Which triggers `buildScope()` again → infinite recursion → StackOverflowError

All test runners fail with this crash:
- `FirLightTreeDiagnosticsWithLatestLanguageVersionTestGenerated`
- `PhasedJvmDiagnosticLightTreeTestGenerated`
- `PhasedJvmDiagnosticPsiTestGenerated`
- `LLDiagnosticsFirTestGenerated`
- `LLPartialDiagnosticsFirTestGenerated`
- `LLReversedDiagnosticsFirTestGenerated`

The expected behavior would be a proper diagnostic (e.g., `CYCLIC_INHERITANCE_HIERARCHY`) instead of a compiler crash. The bug is reproduced in FIR/K2 — the compiler still crashes on this input.
