# KT-36996

## Summary
Wrong `OPERATOR_MODIFIER_REQUIRED` error for Delegate class when class has valid delegate special extension functions.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt36996.kt`

## Observations
The test code uses a `Delegate` class with member `getValue`/`setValue` functions that lack the `operator` modifier,
alongside top-level extension functions with `operator` modifier, and local extension functions inside a class.
The K1 compiler incorrectly reported `OPERATOR_MODIFIER_REQUIRED` because it would pick the non-operator member
functions as candidates and only then check for the operator modifier, ignoring the valid operator extension functions.

In the FIR/K2 compiler, the code compiles successfully (`RUN_PIPELINE_TILL: BACKEND`) with no diagnostics produced.
The K2 frontend correctly resolves the delegate to the extension functions that have the `operator` modifier,
so `OPERATOR_MODIFIER_REQUIRED` is not emitted. The bug described in the ticket is not present in K2.
