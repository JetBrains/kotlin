# KT-48183

## Summary
`DeprecatedByOverridden` deprecation message contains a non-i18n (hardcoded) string in `compiler/frontend/src/org/jetbrains/kotlin/resolve/deprecation/Deprecation.kt` line 132.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt48183.kt`

## Observations
The test triggers the `OVERRIDE_DEPRECATION` diagnostic in FIR by creating a class that overrides a deprecated interface method without marking the override as deprecated itself. The `OVERRIDE_DEPRECATION` warning is produced by `FirOverrideChecker` (line 305) and its message is registered in `FirErrorsDefaultMessages.kt` using the i18n message map mechanism.

The test passes with the `OVERRIDE_DEPRECATION` diagnostic marker on `Derived.foo()`, confirming that:
1. The scenario described in the ticket (overriding a deprecated member) is reproducible
2. FIR produces the `OVERRIDE_DEPRECATION` warning for this case

The original K1 issue (non-i18n string in `Deprecation.kt` at line 132 for `DeprecatedByOverridden`) remains open per YouTrack. The FIR equivalent produces the diagnostic but defines its message through the proper i18n message map in `FirErrorsDefaultMessages.kt`.
