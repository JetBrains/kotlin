# KT-18967

## Summary
Visibility error "cannot access 'InlineOnly': it is internal in 'internal'" is reported twice on the same annotation.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt18967.kt`

## Observations
The FIR compiler reports `INVISIBLE_REFERENCE` twice for `@kotlin.internal.InlineOnly` on an inline function.
The diagnostic marker `<!INVISIBLE_REFERENCE, INVISIBLE_REFERENCE!>InlineOnly<!>` was automatically generated by the test framework on the first run, confirming that the K2/FIR compiler still produces two `INVISIBLE_REFERENCE` diagnostics for the annotation — once when resolving the annotation type and once when resolving the annotation call.

The test passes by asserting the current (broken) behavior: two `INVISIBLE_REFERENCE` diagnostics where one would suffice.
