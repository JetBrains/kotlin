# KT-80047

## Summary
Type Inference: avoid creating a type variable for the return type of anonymous function with specified return type.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt80047.kt`

## Observations
The test demonstrates the current behavior where anonymous functions with explicitly specified return types
are used in generic inference contexts (e.g., `apply(fun(): String = "hello")`). The ticket notes that
a type variable `_R` is currently created for such return types even though they are explicitly stated,
whereas the desired behavior would be to eliminate this unnecessary type variable creation.

The test compiles successfully with `RUN_PIPELINE_TILL: BACKEND`, confirming that the issue does not
cause compilation failures but is a potential optimization/correctness improvement in the type inference
engine. The current behavior (creating `_R` type variable even for explicitly-typed anonymous functions)
is captured by the test, making the bug reproducible for verification when the fix is implemented.

The test required manually adding the `GENERATED_FIR_TAGS` comment since `--update-test-data` did not
write it automatically for the `PhasedJvmDiagnosticLightTreeTestGenerated` and
`PhasedJvmDiagnosticPsiTestGenerated` test classes. After manually adding the tags, all tests passed.
