# KT-12617

## Summary
Cannot infer types in lambda parameter if multiple overloads are available.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt12617.kt`

## Observations
When two overloads of a function share the same first parameter type `(T) -> K`, the FIR compiler
fails to infer the type of the lambda parameter `it` when calling with only that first argument.

The test shows:
- `groupFoldBy` call produces `NONE_APPLICABLE` and `NO_VALUE_FOR_PARAMETER` diagnostics
- `it` inside the lambda produces `UNRESOLVED_REFERENCE` — the type of the implicit lambda parameter
  cannot be inferred because the overload resolution is ambiguous at that point

This confirms the bug is still present in K2/FIR: type inference fails for lambda parameters when
multiple overloads match and it's not yet clear which one to pick.
