# KT-2880

## Summary
Confusing error message when trying to assign to a non-mutable map using index operator.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt2880.kt`

## Observations
When the code `map["a"] = true` is used on a `Map<String, Boolean>` (non-mutable), K2/FIR reports `NO_SET_METHOD` on the index expression `["a"]`. The diagnostic marker `map<!NO_SET_METHOD!>["a"]<!> = true` confirms the bug is present: the compiler emits `NO_SET_METHOD` but the error message does not indicate which class is missing the set method, making it confusing. A comment in the ticket confirms "K2 reports only `NO_SET_METHOD`". The test passes with the diagnostic marker asserting the current (broken/confusing) behavior.
