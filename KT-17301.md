# KT-17301

## Summary
Long chained expressions compilation fails with StackOverflowError

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt17301.kt`

## Observations
The test triggers a `java.lang.StackOverflowError` in the FIR (K2) compiler when processing 500
chained method calls (`.foo()` repeated 500 times). This confirms the bug is present in K2/FIR
as well as the original K1 frontend.

The test run produced two failures:
1. `java.lang.StackOverflowError` — the exact bug described in the ticket, occurring during FIR resolution
   of the deeply nested dot-qualified expression chain.
2. `AssertionFailedError: Actual data differs from file content: kt17301.kt` — because the StackOverflowError
   interrupted the test framework before it could write the `GENERATED_FIR_TAGS` directive to the file.

The StackOverflowError appeared in both:
- `PhasedJvmDiagnosticLightTreeTestGenerated` (compiler/fir/analysis-tests)
- `LLDiagnosticsFirTestGenerated` (analysis/low-level-api-fir)

The `GENERATED_FIR_TAGS` were not written to the `.kt` file because the compilation crashed before
the framework could complete. The bug is the same deep recursion issue, but now occurring in the
FIR frontend rather than K1's `ControlFlowProcessor`.
