# KT-33247

## Summary
Bad performance when editing Kotlin code with complex type inference and overloaded function calls.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt33247.kt`

## Observations

The ticket describes severe IDE performance degradation when editing Kotlin files in IntelliJ IDEA
2019.2, specifically with the "new type inference algorithm" enabled. YourKit profiling pointed to
hot spots in `CallResolver.computeTasksAndResolveCall` and `ExpressionTypingVisitorDispatcher.getTypeInfo`
— components of the K1 (legacy) frontend.

The test creates minimal Kotlin code with multiple overloaded `process()` functions and complex call
chains that exercise type inference machinery, similar to what was described in the ticket.

The FIR (K2) analysis test passes cleanly without any diagnostics or errors. Since K2/FIR is the
current Kotlin frontend and has replaced the K1 inference engine where the performance issue was
rooted, the issue is no longer reproducible in FIR: the code compiles correctly and without the
performance problems described in the K1 frontend.

`repro: false` — the issue was in the K1 type inference engine, which FIR no longer uses. The
described performance problem is not observable in the FIR compiler pipeline.
