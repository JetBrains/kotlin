# KT-13398

## Summary
Safe call on a generic type with nullable upper bound doesn't make it possible to call javaClass extension.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt13398.kt`

## Observations
In K1 compiler, the following code failed with `UNRESOLVED_REFERENCE_WRONG_RECEIVER`:
```kotlin
fun <T> describe(t: T) {
    println(t?.javaClass)
}
fun <T : Any?> describe2(t: T) {
    println(t?.javaClass)
}
```
The error occurred because the safe call `t?.javaClass` on an unconstrained (nullable upper bound) type parameter
did not properly narrow the receiver type, so the `javaClass` extension (defined for `Any`) could not be resolved.

In K2/FIR, both functions compile successfully with `RUN_PIPELINE_TILL: BACKEND` and no diagnostic errors.
The test passes cleanly, confirming the issue is fixed in the K2 compiler. The ticket is also tagged
`not-reproducible-in-k2`.
