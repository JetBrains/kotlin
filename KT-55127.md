# KT-55127

## Summary
Ensure correctness of `NewConstraintSystemImpl#fixVariable` — constraints containing a fixing type variable are removed from other variables instead of being substituted with the fixed type, potentially relaxing bounds incorrectly.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt55127.kt`

## Observations
The ticket describes an internal concern in the type inference constraint system: when fixing a type variable (e.g., A → Int), constraints on other type variables that reference A are removed rather than substituted (e.g., `B ≥ A` is removed instead of becoming `B ≥ Int`). This could incorrectly relax bounds for further variables.

The test exercises this code path with multiple interrelated type variables:
- `coerceIn<A, B : A>` — A fixed from explicit type annotation, B constrained by A
- `chain3<A, B : A, C : B>` — three-level type variable chain
- `transform<T, R>` — standard higher-order function with dependent type variables

The test compiles and passes cleanly without any diagnostic errors. The compiler correctly handles these scenarios — no incorrect type inference or spurious errors were observed. This suggests the described issue (if it existed) has already been addressed, or the observable behavior was never incorrect for these patterns.

Since the test code behaves as expected (correct inference, no wrong diagnostics), the bug described in the ticket — relaxed bounds due to constraint removal rather than substitution — is not observable in the current compiler. The issue is `repro: false`.
