# KT-19189

## Summary
AssertionError: Recursion detected on input when a subclass has a method with the same name as a method in the parent class, and the parent class method creates an instance of the subclass and calls that method on it.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt19189.kt`

## Observations
The test reproduces the structural pattern from the ticket: `ComponentBuilder` has a `keybind(String, ...)` method that creates a `KeybindComponentBuilder` instance and calls `.keybind(key)` on it. `KeybindComponentBuilder` extends `ComponentBuilder` and also declares its own `keybind(String)` method. In the K1 frontend this caused an `AssertionError: Recursion detected on input`, but in K2/FIR the code compiles without errors or warnings. The test passes cleanly with `RUN_PIPELINE_TILL: BACKEND`, indicating the issue is fixed in the FIR frontend.
