# KT-18892

## Summary
False negative USELESS_CAST for smartcasted variable with safe cast (`as?`) and safe call — no warning was reported for `(a as? CharSequence)?.length` even when `a` was already smartcasted to `CharSequence`.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt18892.kt`

## Observations
The K2/FIR compiler correctly reports `USELESS_CAST` for **both** forms of the cast:
- `(a as CharSequence).length` → `USELESS_CAST` ✓
- `(a as? CharSequence)?.length` → `USELESS_CAST` ✓

In K1, the second case (safe cast `as?`) was a false negative — no warning was produced. The comment in the ticket notes this regression was introduced in Kotlin 1.1.3-2. In K2/FIR, the issue no longer exists: both useless casts are correctly diagnosed. The test passes cleanly with both diagnostic markers present.
