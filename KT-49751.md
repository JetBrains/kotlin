# KT-49751

## Summary
False negative USELESS_CAST with interfaces: when a variable of interface type `I` is smart-cast inside `if (x is K)`, the redundant cast `x as K` is not reported as USELESS_CAST.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt49751.kt`

## Observations
The test reproduces the false negative. In the test:
- `foo(x: I)`: inside `if (x is K)`, the cast `x as K` is NOT reported as `USELESS_CAST` (false negative — the bug is present).
- `foo2(x: Any)`: inside `if (x is K)`, the cast `x as K` IS correctly reported as `USELESS_CAST`.

The test passes by asserting the current broken behavior: no diagnostic is emitted for `x as K` when `x` is of interface type, even though the smart cast makes the cast unnecessary. The bug is still present in FIR.
