# KT-10443

## Summary
Missing type inference error on variable setter: `y.proxyT = 5` should be an error when `y: Y<out Number>`, but K1 produced no diagnostic.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt10443.kt`

## Observations
K2/FIR correctly reports errors for all three problematic cases in `test2(y: Y<out Number>)`:

- `y.t = 5` → `ASSIGNMENT_TYPE_MISMATCH` (was already reported in K1)
- `y.setter(5)` → `MEMBER_PROJECTED_OUT` (was already reported in K1)
- `y.proxyT = 5` → `SETTER_PROJECTED_OUT` (was **missing** in K1, now correctly reported in K2)

The issue is not reproducible in K2/FIR — `SETTER_PROJECTED_OUT` is properly emitted for the extension property setter call. The ticket is tagged "not-reproducible-in-k2", which matches this finding. The test passes cleanly with `RUN_PIPELINE_TILL: FRONTEND` (required because the FIR errors prevent backend compilation).
