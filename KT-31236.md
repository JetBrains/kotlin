# KT-31236

## Summary
Prohibit implicit intersection types as return types in new inference

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt31236.kt`

## Observations
The test code:
```kotlin
interface A
interface B

fun foo(x: Any?) = if (x is A && x is B) x else null
```

With FIR/K2, the function `foo` infers an intersection type `(A & B)?` as its return type (the `GENERATED_FIR_TAGS` includes `intersectionType`). The compiler accepts this code without any error or warning, even though the ticket requests that implicit intersection types as return types should be prohibited.

The test passes cleanly, but with no diagnostic markers — meaning the compiler silently allows the implicit intersection return type rather than emitting an error. This confirms the bug described in the ticket is still present.
