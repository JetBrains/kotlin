# KT-11491

## Summary
Meta issue tracking Type Checks and Casts diagnostics — specifically improving diagnostics for unchecked cast (KT-5535) and useless cast (KT-33403).

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt11491.kt`

## Observations
The test exercises two cast diagnostic scenarios:

1. **Useless cast** (`USELESS_CAST`): `42 as Int` and `"hello" as String` — casting a value to its own type.
2. **Unchecked cast** (`UNCHECKED_CAST`): `list as List<Int>` where `list: List<Any>` — cast where generic type parameter cannot be verified at runtime due to type erasure.

Both diagnostics are correctly produced by the K2/FIR compiler. The test passes cleanly without `--update-test-data`, confirming these diagnostics are already implemented and functioning.

**Conclusion:** The core behaviors described in KT-11491 (USELESS_CAST and UNCHECKED_CAST diagnostics) are present and working in the FIR compiler. The issue is not reproduced — the fix is already in place.

Note: KT-11491 is a meta/umbrella issue; some of its subtasks (e.g., KT-33403) remain open for more specific edge cases not covered by this basic reproducer.
