# KT-41383

## Summary
Impossible to use callable reference to extension function as default parameter value — the reference was inferred as `KFunction2<Int, Int, Int>` instead of `(Int) -> Int`.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt41383.kt`

## Observations
The test code from the ticket:
```kotlin
fun Int.plusWithDefault(i: Int = 0) = this + i
fun plusRef(ref: (Int) -> Int = Int::plusWithDefault) = ref(5)

fun works() = plusRef(Int::plusWithDefault)
```

In K2/FIR, this compiles without any diagnostic errors. The test passes cleanly with `RUN_PIPELINE_TILL: BACKEND`, confirming that the type inference correctly unifies `Int::plusWithDefault` with the expected type `(Int) -> Int` even when used as a default parameter value. The bug described in the ticket (type mismatch error) is no longer present in FIR — the issue has been fixed in K2.
