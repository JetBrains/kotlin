# KT-37783

## Summary
MPP Java Interop: Extra functions declared in actual super classes can't be resolved in Java for a common child class.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt37783.kt`

## Observations
The test reproduces the FIR analysis behavior described in the ticket. When `expect`/`actual` declarations are present, the FIR frontend cannot resolve extra methods defined only on the `actual` class when calling through either the `expect` type or a common child class:

- `MPPSuper().hello()` → `<!UNRESOLVED_REFERENCE!>hello<!>`
- `CommonChild().hello()` → `<!UNRESOLVED_REFERENCE!>hello<!>`

The test framework also detected `<!NOT_A_MULTIPLATFORM_COMPILATION!>` and `<!EXPECT_AND_ACTUAL_IN_THE_SAME_MODULE!>` diagnostics on the `expect`/`actual` declarations (expected for a single-file test context).

The `UNRESOLVED_REFERENCE` on both `hello()` calls confirms that the FIR frontend does not propagate extra methods from an `actual` class to the `expect` type or common subtypes, matching the IDE-side failure described in the ticket (where `CommonChild().hello()` is reported as unresolved in the IDE even though compilation succeeds).
