# KT-33904

## Summary
IntelliJ IDEA frozen while editing Kotlin file — Kotlin compiler freezes (infinite loop) in constraint incorporation when processing complex recursive generic type parameters.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt33904.kt`

## Observations
The ticket reports that the K1 compiler (old frontend) enters an infinite loop in `ConstraintIncorporationKt.generateNewBound` / `incorporateBound` when processing classes with deeply recursive generic type bounds (e.g. `SERVICE : ModuleBasedServiceWithProjectDefault<STATE, SERVICE, APP_SERVICE, MODULE_SERVICE>`).

A minimal reproducer (from the last comment in the ticket) was written using the isolated recursive generic structure. The test compiles successfully with FIR (K2 frontend) without any infinite loop or freeze — the compiler terminates and produces no errors.

The FIR test passes cleanly (`RUN_PIPELINE_TILL: BACKEND`), meaning the K2/FIR frontend handles the recursive constraint incorporation correctly. The bug is **not reproducible** in the current FIR compiler — it was specific to the old K1 constraint system (`ConstraintSystemBuilderImpl`) which has since been replaced.
