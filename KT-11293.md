# KT-11293

## Summary
False warning "No cast needed" (USELESS_CAST) for lambda argument when the cast is necessary to disambiguate between overloaded functions.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt11293.kt`

## Observations
When two overloaded functions `foo` accept different function types (`(Int) -> Double` and `(Int) -> Int`),
the explicit cast `{i: Int -> 1.0} as (Int) -> Double` is necessary to select the correct overload.
However, K2/FIR still incorrectly reports `USELESS_CAST` for this cast expression, even though removing
it would make the call ambiguous.

The test framework inserted `<!USELESS_CAST!>as (Int) -> Double<!>` during the first `--update-test-data`
run, confirming the diagnostic is emitted. The test passes by asserting this (currently incorrect)
compiler behavior.

Note: the YouTrack ticket is tagged "not-reproducible-in-k2", but the FIR diagnostics test shows
that USELESS_CAST is still reported in K2 for this pattern.
