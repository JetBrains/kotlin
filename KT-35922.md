# KT-35922

## Summary
Partial resolve test fails if argument expression type is not rewritten with SmartCastManager

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt35922.kt`

## Observations
The issue references `org.jetbrains.kotlin.idea.resolve.PartialBodyResolveTestGenerated.testSmartCastInTheSameStatement`,
which is a K1/IDE-specific test about partial body resolve failing when smart-cast argument expression types
are not rewritten by SmartCastManager.

The FIR analysis test was created to check various smart-cast-in-argument-position scenarios:
- Smart cast after `is`-check in function call argument (`foo(x)` where `x` was checked `is String`)
- Smart cast after null check via `?:` return
- Smart cast in `when` expression conditions and branches
- Smart cast with `&&` compound conditions
- Explicit `as` cast in argument position

In K2/FIR, all of these cases compile correctly to BACKEND without any errors or crashes.
The test passes cleanly, indicating that the issue with argument expression types not being rewritten
does NOT affect K2/FIR — this was a K1/partial body resolve issue that is not present in the
FIR compiler frontend.
