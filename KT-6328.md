# KT-6328

## Summary
Lookup of short form `invoke()` operator function doesn't work on generic properties — not reproducible in K2.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt6328.kt`

## Observations
The test reproduces the original K1 scenario: a generic extension property `val <T> T.self: T` returning an
invokable type `F<T>` where `F<T>` has an `operator fun invoke()`. In K1, calling `F<String>().self()` failed
with `FUNCTION_EXPECTED` ("Expression 'self' of type 'F<String>' cannot be invoked as a function. The function
'invoke()' is not found"), while the long form `F<String>().self.invoke()` worked fine.

In K2/FIR, all three forms (`self.invoke()`, `self()`, and direct `()`) compile without any errors or
diagnostics. The test passed cleanly without any diagnostic markers, confirming the bug is fixed in K2.
The ticket is tagged "not-reproducible-in-k2", consistent with this result.
