# KT-6071

## Summary
The `USELESS_CAST` warning message says "No cast needed. You can use ':' if you need a cast to a super type." even when the cast is NOT to a super type (the majority of cases). The second phrase is misleading and should be omitted when the cast is not to a super type.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt6071.kt`

## Observations
- The `USELESS_CAST` warning is reproduced for `x as String` (cast to same type, the majority case described in the ticket).
- In FIR/K2, casting to a super type (`x as Any`) does NOT produce `USELESS_CAST` (the diagnostic was removed for this case).
- The test confirms that `USELESS_CAST` is still emitted for non-super-type casts (`as String`), which is exactly the case where the original message text includes the misleading phrase "You can use ':' if you need a cast to a super type."
- The test passes with the `USELESS_CAST` marker asserted on `x as String`, confirming the warning is generated for this scenario.
- The first run updated the diagnostic marker position from `<!USELESS_CAST!>x as String<!>` to `x <!USELESS_CAST!>as String<!>` (framework preference: mark just the `as Type` part).
