# KT-24724

## Summary
`IMPLICIT_CAST_TO_ANY` warning isn't reported in new inference (FIR/K2) when a `when` expression has branches with incompatible return types that get implicitly cast to `Any`.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt24724.kt`

## Observations
The test code uses a `when` expression inside a `map` lambda where the branches produce `Int` (from `it + 1`), `String` (from `"" + ""`), and `Int` (from `else -> 42`) values. The type of the `when` expression is inferred as `{Comparable<*> & java.io.Serializable}` (intersection type), but no `IMPLICIT_CAST_TO_ANY` diagnostic is emitted.

The test passes cleanly with `RUN_PIPELINE_TILL: BACKEND` and no diagnostic markers — confirming the warning is absent in FIR. The K1 compiler (old inference) would report `IMPLICIT_CAST_TO_ANY` here, but the new inference (K2/FIR) silently drops this warning. The generated FIR tags (`intersectionType`, `smartcast`) indicate the compiler does compute an intersection/Any type but does not emit the expected diagnostic.
