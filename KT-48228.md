# KT-48228

## Summary
Ambiguous function resolution with generic parameter and trailing lambda parameter — `test(box) { }` incorrectly produces OVERLOAD_RESOLUTION_AMBIGUITY when one overload takes `P` (a Props-bound type parameter) and the other takes `() -> Unit`.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt48228.kt`

## Observations
The test was created with two overloads (`test(box: Box<P>, props: P)` and `test(box: Box<P>, block: () -> Unit)`) and a call `test(box) { }` using a trailing lambda, which should unambiguously resolve to the second overload.

The test runs with `RUN_PIPELINE_TILL: BACKEND` and no inline diagnostic markers (no `OVERLOAD_RESOLUTION_AMBIGUITY` expected). All test runners (FirLightTree, PhasedJvm, etc.) passed cleanly, meaning the K2/FIR compiler resolves the trailing lambda call to the correct overload without any ambiguity error.

The bug described in the ticket (overload resolution ambiguity reported for `test(box) { }`) is no longer present in the current K2/FIR compiler — the issue appears to have been fixed.
