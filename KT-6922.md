# KT-6922

## Summary
`x is Int` when type of x is Unit should be a warning, not an error.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt6922.kt`

## Observations
The ticket reports that in K1, `x is Int` where `x: Unit` produces an `INCOMPATIBLE_TYPES` error instead of a warning. The desired behavior is a warning (since the check evaluates to `false` at runtime rather than crashing).

In K2/FIR, the same code produces `IMPOSSIBLE_IS_CHECK_ERROR` ("Check for instance is always 'false'"), which is still a **compilation error**, not a warning. The test was created with `// RUN_PIPELINE_TILL: FRONTEND` and the diagnostic marker `<!IMPOSSIBLE_IS_CHECK_ERROR!>x is Int<!>`.

While the specific `INCOMPATIBLE_TYPES` diagnostic from K1 is no longer produced (hence the "not-reproducible-in-k2" tag), the underlying issue — that this should be a warning rather than an error — remains: `IMPOSSIBLE_IS_CHECK_ERROR` is still reported as an error, preventing backend compilation. The test passes by asserting the current (broken) K2 behavior.
