# KT-37606

## Summary
NI: Confusing error message due to missing candidates when trying to mutate read-only Map

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt37606.kt`

## Observations
The ticket describes a confusing error message in the New Inference (K1) engine: when trying to assign
to a read-only `Map` via indexed access (`cache["foo"] = "bar"`), the error incorrectly showed
`StringBuilder.set` as a candidate with a receiver mismatch, instead of showing `MutableMap.set`
or a clear message about the missing operator.

In FIR/K2, the compiler produces the `NO_SET_METHOD` diagnostic on `cache["foo"]`, which clearly
communicates that there is no `set` operator available on the receiver type `Map<String, String>`.
This is the correct and expected behavior — the confusing message described in the ticket is not
present in FIR. The issue appears to have been resolved as part of the K2 compiler implementation.
