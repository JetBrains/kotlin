# KT-18134

## Summary
Warning `IMPLICIT_CAST_TO_ANY` isn't reported when an expression involving mixed types is used as a function parameter.

## Test file
`compiler/fir/analysis-tests/testData/resolve/problems/kt18134.kt`

## Observations
The ticket describes a K1 inconsistency where `IMPLICIT_CAST_TO_ANY` is reported for some expressions (variable assignment, string templates) but not when used as function parameters.

In K2/FIR, the bug is even more severe: `IMPLICIT_CAST_TO_ANY` is not reported in ANY of the described cases:
- `val v = if (true) A() else 1` — no warning (K1 DID report it here)
- `print("${if (true) A() else 1}")` — no warning (K1 DID report it here)
- `foo(if (true) A() else 1)` — no warning (K1 also didn't report it here)
- `print(if (true) A() else 1)` — no warning
- `1 to if (true) A() else 1` — no warning

The test passes because there are no diagnostic markers (FIR produces no `IMPLICIT_CAST_TO_ANY` diagnostics), confirming the bug is reproduced — the warning is entirely absent in K2/FIR for all these patterns.
